# MAX Order Bot

Бот для MAX на Java с `long polling`, `SQLite`, админ-панелью, сценарием заказа и запуском через Docker.

## Что умеет

- `long polling` через `GET /updates`
- заказ по шагам: салат -> суп -> горячее -> доп. позиция -> ФИО -> подтверждение
- админ-панель с несколькими администраторами
- список активных пользователей с пагинацией
- заявки за текущий день по МСК с пагинацией
- добавление администраторов по ID
- управление позициями меню через бота
- все кнопки сделаны с эмодзи
- хранение данных в `SQLite`

## Переменные окружения

- `MAX_BOT_TOKEN` - токен бота из MAX
- `INITIAL_ADMIN_IDS` - список ID администраторов через запятую, например `123,456`
- `MAX_API_BASE_URL` - API MAX, по умолчанию `https://platform-api.max.ru`
- `SQLITE_PATH` - путь к базе, по умолчанию `./data/bot.db`
- `POLL_TIMEOUT_SECONDS` - timeout long polling, по умолчанию `30`
- `POLL_LIMIT` - размер пачки обновлений, по умолчанию `100`

## Локальная сборка

```bash
mvn -DskipTests package
java -jar target/max-order-bot-1.0.0.jar
```

## Docker build

```bash
docker build -t max-order-bot .
```

## Docker run

```bash
mkdir -p /opt/max-order-bot/data

docker run -d \
  --name max-order-bot \
  --restart unless-stopped \
  -e MAX_BOT_TOKEN="ваш_токен" \
  -e INITIAL_ADMIN_IDS="123456789" \
  -v /opt/max-order-bot/data:/app/data \
  max-order-bot
```

## Как это работает

### Старт

- `/start` или текст `Начать` показывает приветствие
- пользователь видит кнопку `🍽️ Сделать заказ`
- если пользователь администратор, дополнительно видит `🛠️ Админ панель`

### Сценарий заказа

1. выбор салата
2. выбор супа
3. выбор горячего
4. выбор доп. позиции
5. ввод ФИО
6. подтверждение заказа

После подтверждения:

- заявка сохраняется в `SQLite`
- уведомление уходит всем администраторам
- в разделе `📦 Заявки` она видна только в текущий день по МСК

### Админ-панель

1. `👥 Все пользователи`
2. `➕ Добавить админа`
3. `📦 Заявки`
4. `🍽️ Изменить позиции`
5. `📝 Перейти к опросу`

## Примечания по MAX API

Бот использует:

- `GET /updates` с типами `message_created,message_callback`
- `POST /messages?user_id=...`
- `POST /answers?callback_id=...`

Именно такой формат описан в актуальной документации MAX и в их открытой OpenAPI-схеме.
